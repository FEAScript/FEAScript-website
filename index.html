FEAScript is a lightweight, open-source finite element simulation library developed in JavaScript. It allows users to create and run simulations for physics and engineering problems in both browser-based and server-side environments. FEAScript serves as an excellent tool for building interactive web applications and facilitates hands-on learning of computational mechanics by providing a versatile platform to explore and solve complex problems. Our goal is to democratize finite element analysis by making simulation capabilities accessible to everyone, everywhere.

Getting Started

There are two ways to create simulations with FEAScript:

  • JavaScript API (FEAScript Core): For developers comfortable with coding, the JavaScript API provides full programmatic control. It runs in three environments: in the browser with a simple HTML page, via Node.js for server-side applications, or with online interactive notebooks (e.g. Scribbler ).
  • Visual Editor (FEAScript Platform): The FEAScript Platform provides a no-code, block-based interface (based on Blockly ) for creating simulations without writing any JavaScript code. With the FEAScript Platform, you can connect visual blocks, create complex models, and save or load your projects in XML format for easy sharing and reuse.

Quick Start: To use FEAScript in your HTML, include it from our hosted build (i.e., import { FEAScriptModel } from "https://core.feascript.com/dist/feascript.esm.js";) or download it from GitHub → add a canvas (e.g., <div id="solutionPlot"></div>) → add a mesh file (e.g., your.msh from Gmsh ) or use FEAScript mesh generation tools → create and run a simulation using the JavaScript API. Here is a simple example (see tutorials for more details): 

<body>
  <!-- ...body region... -->
  <script type="module">
  // Import FEAScript library
  import { FEAScriptModel, importGmshQuadTri, plotSolution } from "https://core.feascript.com/dist/feascript.esm.js";

  window.addEventListener("DOMContentLoaded", async () => {
    // Load and parse Gmsh mesh
    const meshContent = await (await fetch("path/to/your.msh")).text();
    const meshFile = new File([meshContent], "your.msh");
    const parsedMesh = await importGmshQuadTri(meshFile);

    // Create and configure model
    const model = new FEAScriptModel();
    model.setSolverConfig("solverType"); // e.g., "solidHeatTransferScript"
    model.setMeshConfig({parsedMesh, meshDimension: "2D", elementOrder: "linear"});

    // Apply boundary conditions (Gmsh physical group tags)
    model.addBoundaryCondition("boundaryIndex", ["conditionType", /* parameters */]);

    // Solve
    const { solutionVector, nodesCoordinates } = model.solve();

    // Plot results
    plotSolution(solutionVector, nodesCoordinates, model.solverConfig, model.meshConfig.meshDimension, "contour", "solutionPlot", "unstructured");
  });
  </script>
  <!-- ...continue of body region... -->
</body>

The above code is an illustrative example of running FEAScript. Adapt paths, solver types, and boundary conditions to your specific problem.

Features

The following list highlights key FEAScript features:

  • 🧠 Physical Modeling
    • Solid heat conduction solver
    • Front propagation solver
  • 📐 Geometry and Meshing
    • Simple mesh generation (1D and rectangular 2D domains)
    • Unstructured mesh import from Gmsh (.msh file format)
  • ⚙️ Numerical Methods
    • Linear system solvers (LU decomposition, Jacobi method)
    • Newton-Raphson method for nonlinear systems
  • 📊 Performance & Parallelization
    • Web worker support for multi-threaded computation
  • 📈 Visualization & Post-Processing
    • Interactive line and surface visualization with Plotly
  • 🚀 Platform & Deployment
    • Browser runtime (ES modules)
    • Node.js runtime (npm )
    • Online notebooks integration (e.g. Scribbler )
    • No-code interface with FEAScript Platform visual editor
